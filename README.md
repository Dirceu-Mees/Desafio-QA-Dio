# Desafio-Dio

Projeto de Testes: Fluxo de Pagamento via Boleto - E-commerce
📝 Descrição do Projeto
Este projeto foi desenvolvido como parte do desafio de QA da DIO (Digital Innovation One). O objetivo foi aplicar conceitos de planejamento e execução de testes em um cenário real de e-commerce, focando especificamente nas regras de negócio para o meio de pagamento Boleto Bancário.

🛠️ Ferramentas Utilizadas
Jira Software: Gestão do projeto e organização das tarefas.

Zephyr Scale: Gestão técnica de testes, escrita de casos de teste e ciclos de execução.

🎯 Estratégia de Testes
Para garantir a cobertura completa das regras de negócio, foram aplicadas duas metodologias de escrita:

1. BDD (Behavior Driven Development)
Utilizado para validar comportamentos esperados do sistema sob a perspectiva do usuário, utilizando a sintaxe Gherkin (Dado, Quando, Então).

CT01: Sucesso no parcelamento em até 3x no boleto.

CT02: Bloqueio de tentativa de parcelamento acima de 3x.

2. Step-by-Step (Manual Tradicional)
Utilizado para validações técnicas e de interface que exigem passos manuais específicos.

CT03: Validação de valor mínimo de parcela (R$ 100,00).

CT04: Validação de legibilidade e geração de PDF do boleto.

📊 Resultados e Evidências
O ciclo de teste foi executado com 100% de aproveitamento, garantindo que todas as funcionalidades planejadas estão operando conforme os critérios de aceitação.

Ciclo de Execução: PQD-R2 - Sprint Pagamento Boleto

Status Final: Concluído (Sucesso em todos os cenários).
